# Workflow: Website Creation

## Objective
Design and build conversion-optimized websites for clients across industries.

## Required Inputs
- Client name, industry, and business type (local, e-commerce, SaaS, personal brand, etc.)
- Website goal (leads, sales, bookings, portfolio, info)
- Pages needed (home, about, services, contact, blog, shop, etc.)
- Preferred tech stack (or let agent recommend)
- Budget tier (landing page / multi-page / full web app)
- Competitor sites they like/dislike
- Brand assets (logo, colors, fonts) — if none, flag for brand creation first

## Agent Roles
- **Queen (Orchestrator)**: Manages project, coordinates agents, quality checks
- **Researcher**: Competitor analysis, industry best practices, conversion benchmarks
- **Architect**: Site structure, page hierarchy, user flow, tech stack recommendation
- **Copywriter**: All website copy — headlines, body text, CTAs, meta descriptions
- **Developer**: Builds the site (HTML/CSS/JS or framework per stack choice)
- **SEO Agent**: On-page SEO, page speed, meta tags, schema markup
- **QA Agent**: Cross-browser testing, mobile responsiveness, link checking

## Process
1. Queen receives brief → scopes project → assigns agents
2. Researcher delivers competitor analysis and conversion insights
3. Architect designs site map and wireframe structure
4. Copywriter writes all copy before any code is written
5. Developer builds pages using approved copy and structure
6. SEO Agent optimizes all pages
7. QA Agent tests and reports issues
8. Queen delivers final package

## Output Format
- Complete website files in `deliverables/websites/[client-name]/`
- Deployment-ready (Netlify / Vercel / cPanel depending on client)
- SEO report
- Handoff doc with login credentials and update instructions

## Edge Cases
- E-commerce sites require payment gateway discussion before starting
- GDPR/privacy policy required for all EU-targeting sites
- If client has no hosting → recommend and document setup steps
